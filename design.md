# Mystic Tarot — 디자인 시스템

Notion 다크 모드에서 느껴지는 "거의 블랙에 가까운 배경 + 절제된 색 사용" 톤을 기준으로 삼는다.
이 문서에 없는 색/버튼/여백 값을 새로 만들지 말고, 항상 아래 팔레트 안에서만 조합한다.
`index.html`의 `:root` 변수가 이 문서의 원본(source of truth)이며, 문서와 코드는 항상 같은 값을 가리켜야 한다.

## 1. 배경

| 변수 | 값 | 용도 |
|---|---|---|
| `--bg` | `#06000f` | 전체 페이지 배경. 순수 블랙(`#000`)이 아니라 아주 약간 보라 톤을 준 블랙 — Notion 다크모드처럼 눈이 편안한 블랙. |
| `--surface` | `#0f0720` | 배경 위에 얹는 1단계 표면(모달 배경 그라디언트 등). |
| `--card-bg` | `#130928` | 카드/박스형 컴포넌트의 기본 배경. |

배경은 항상 블랙 계열 하나만 쓴다. 화이트/라이트 테마는 만들지 않는다.

## 2. 텍스트 3색 규칙

본문에 실제로 읽히는 텍스트는 아래 3가지 색만 쓴다. 새 화면을 만들 때 4번째 텍스트 색이 필요하다고 느껴지면, 그건 대개 "굵기(font-weight)"나 "크기"로 해결해야 할 문제다.

| 순서 | 변수 | 값 | 용도 |
|---|---|---|---|
| 1 | `--text` | `#e9d5ff` | Primary — 본문, 제목, AI 해석 텍스트, 강조 문장. 화면의 기본 글자색. |
| 2 | `--text-muted` | `#a78bfa` | Secondary — 라벨, 캡션, 보조 설명, placeholder, 비활성 상태. |
| 3 | `--text-accent` | `#fbbf24` (gold) | Accent — 지금 주목해야 할 것. 스트릭 숫자, 가격, 강조 배지, 완료·CTA성 문구. 화면당 꼭 필요한 곳에만 아껴서 쓴다. |

`--lilac`, `--pink`, `--text2`, `--muted`, `--gold`는 과거 코드와의 호환을 위해 남겨둔 별칭이며 각각 위 3색 중 하나를 가리킨다(`--lilac`→`--text`, `--pink`/`--gold`→`--text-accent`, `--text2`/`--muted`→`--text-muted`). **새 코드는 별칭이 아니라 `--text` / `--text-muted` / `--text-accent`를 직접 쓴다.**

### 예외 (본문 텍스트가 아닌 것들)

3색 규칙은 "읽는 텍스트"에 대한 규칙이다. 아래 둘은 텍스트처럼 보여도 실제로는 컴포넌트의 일부라서 예외로 둔다.

1. **소셜 로그인 브랜드 버튼** — 카카오(`#3A1D1D` on `#FEE500`), 네이버(`#fff` on `#03C75A`), 구글(`#333` on `#fff`)은 각 플랫폼 브랜드 가이드라인이 강제하는 색이라 그대로 둔다. 이 3개 버튼 밖에서는 이 색들을 쓰지 않는다.
2. **상태 배지(칩) 색** — `--red`(`#f87171`) / `--green`(`#4ade80`) / `--orange`(`#fb923c`)는 "오늘의 카드" 할 것/하지 말 것 태그, 유사 질문 경고문 강조처럼 아주 짧은 라벨에만 쓴다. 문단 텍스트에는 절대 쓰지 않는다.
3. **버튼 전경색(흰색)** — `.btn-primary`, 소셜 버튼, 배지(pill) 등 색이 채워진 배경 위의 글자는 항상 `#fff`. 이건 "버튼 컴포넌트"의 일부이지 본문 텍스트가 아니므로 3색 카운트에 넣지 않는다.
4. **카드 수트 색(`SC`)** — 완드🔥주황(`#f97316`) / 컵💧파랑(`#60a5fa`) / 소드⚔️보라(`#a78bfa`) / 펜타클🪙초록(`#34d399`) / 메이저 골드(`#fbbf24`)는 뽑힌 카드를 표시하는 아주 작은 칩(`mini-card`)에서 카드 종류를 구분하는 용도로만 쓴다.

## 3. 폰트

```
--font: 'Segoe UI','Apple SD Gothic Neo','Noto Sans KR',sans-serif;
```

시스템 폰트 하나만 쓰고, 어디에서도 다른 `font-family`를 지정하지 않는다(숫자 로마자 표기 등 카드 아트 내부의 `Georgia,serif`처럼 카드 그래픽 자체에 속한 것은 예외).

### 타입 스케일

| 용도 | 크기 | 굵기 |
|---|---|---|
| 브랜드 워드마크 | 25px | 900 |
| 화면 타이틀 | 18–20px | 700 |
| 섹션 제목 / 카드 안 제목 | 14–15px | 700 |
| 본문 / 해석 텍스트 | 13–13.5px | 400, `line-height:2` |
| 라벨 / 캡션 | 10–12px | 400–600 |
| 버튼 텍스트 | 13–15px | 600–700 |

브랜드 워드마크(`MYSTIC TAROT`)의 보라·핑크 그라디언트 텍스트만 유일한 예외다 — 로고 자체의 그래픽 처리이며 3색 규칙 대상이 아니다.

## 4. 버튼

버튼은 아래 2종류만 쓴다. 새 화면에 새로운 버튼 스타일을 만들지 않는다.

- **`.btn-primary`** — 화면당 원칙적으로 1개. `linear-gradient(135deg, var(--purple), var(--violet))` 채움, `border-radius:14px`, `padding:14px`, 흰색 굵은 글자(15px/700), `box-shadow:0 4px 20px #7c3aed44`. "지금 가장 중요한 행동"에만 쓴다(질문하기, 해석 받기, 결제하기 등).
- **`.btn-ghost`** — 보조 행동. 투명 배경 + `1px solid #4c1d9566` 테두리, `border-radius:12px`, `padding:12px`, `--text-muted` 글자색.

### 예외 컴포넌트

- **소셜 로그인 버튼(`.social-btn`)** — `border-radius:12px`로 시스템과 통일하되, 배경/글자색은 §2의 브랜드 예외를 따른다. 카카오/네이버/구글 외 다른 버튼에 이 색을 쓰지 않는다.
- **광고/결제 버튼**(§6, §7)도 위 두 버튼 클래스 중 하나를 그대로 재사용한다. 광고 CTA는 `.btn-ghost` 톤에 가깝게, 결제·구독처럼 돈이 오가는 액션은 `.btn-primary`를 쓴다.

## 5. 여백 · 반경 규칙

4px를 기준 단위로 삼는다: `4 · 8 · 12 · 14 · 16 · 18 · 20 · 24 · 32`.

- **페이지 좌우 거터**: `.shell{padding:0 18px}` — 모든 화면 공통, 절대 바꾸지 않는다.
- **컴포넌트 내부 padding**: 카드형 박스 14–20px, 버튼 12–14px, 칩/배지 4–6px.
- **요소 간 간격(gap/margin)**: 같은 그룹 안 8–10px, 그룹과 그룹 사이 14–16px.

### border-radius 스케일

| 반경 | 대상 |
|---|---|
| 8px | 작은 칩(`mini-card`, `ad-btn`, 카드 슬롯 라벨) |
| 12px | input, `.btn-ghost`, 소셜 버튼, 저널 아이템, stat-box |
| 14px | `.btn-primary`, 스프레드 선택 아이템 |
| 16–18px | `.card-box`, `.interpret-box`, 데일리 카드, 프로필 히어로 |
| 20–24px | 모달·바텀시트(`jm-box`, `warn-box`, `interstitial`) |

## 6. 광고 배치 규칙

- 배너형 광고(`.banner-ad`)는 홈/일지/프로필 화면에서 콘텐츠 사이 한 자리에만, 최대 1개까지 배치한다.
- 광고 CTA 버튼은 `.btn-ghost` 톤의 작은 pill(`.ad-btn`, `border-radius:8px`)만 쓰고 `.btn-primary`와 시각적으로 구분되게 한다 — 사용자가 "결제/핵심 행동"과 "광고"를 혼동하지 않아야 한다.
- 페이월의 "광고 보고 보기" 버튼은 예외적으로 `.btn-primary`를 쓰되, 결제 버튼과 같은 화면에서는 결제 버튼을 먼저(위쪽) 배치해 우선순위를 준다.
- 자세한 AdSense 연동 방법은 [ADS_PAYMENTS_GUIDE_KO.md](ADS_PAYMENTS_GUIDE_KO.md) 참고.

## 7. 결제 버튼 규칙

- 결제/구독 CTA는 항상 `.btn-primary`를 그대로 쓴다. 별도의 "결제 전용 버튼 스타일"을 만들지 않는다.
- 가격 표기는 `--text-accent`(gold)만 사용한다 — 본문 텍스트 색으로 가격을 쓰지 않는다.
- 자세한 TossPayments 연동 방법은 [ADS_PAYMENTS_GUIDE_KO.md](ADS_PAYMENTS_GUIDE_KO.md) 참고.

## 8. 앞으로 새 화면을 추가할 때

1. 새 텍스트 색이 필요하다고 느껴지면 먼저 `--text` / `--text-muted` / `--text-accent` 중 하나로 표현할 수 없는지 확인한다.
2. 새 버튼이 필요하면 `.btn-primary` / `.btn-ghost` 중 하나를 재사용할 수 없는지 먼저 확인한다.
3. 여백/반경은 위 스케일 표에서 가장 가까운 값을 쓴다. 표에 없는 값(예: `15px`, `19px`)을 임의로 쓰지 않는다.
4. 배경은 항상 `--bg`/`--surface`/`--card-bg` 3단계 안에서만 조합한다.
